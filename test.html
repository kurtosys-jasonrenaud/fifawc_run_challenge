Before you start

Local testing only. Do not commit client secrets or access tokens. This file is excluded from the public GitHub Pages deploy. Register an app at strava.com/settings/api and set the callback domain to match where you host this file locally.

  • OAuth authorize works in the browser (redirect to Strava and back with a code).
  • Token exchange needs your client secret. Do that locally or via a tiny backend. Never commit the secret or ship it in production JavaScript.
  • Activity API calls are often blocked by CORS from a static page. If fetch fails, use the pasted access token with a local proxy or serverless function.

Recommended production path

  • Static GitHub Pages can host OAuth redirect pages, but not secret handling.
  • Small backend (Cloudflare Worker, GitHub Action, or Node server) exchanges codes, stores refresh tokens, and proxies Strava API calls to avoid CORS.
  • Per-runner linking each athlete authorizes once; your backend keeps their refresh token and syncs new runs into leaderboard.json or a database.
  • Scopes request activity:read_all if runners use private activities.
